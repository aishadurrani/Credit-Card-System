Core Concept

Proposed Steps:


1. Create modules / Business logic in java using eclipse. User can only use console for input and output.


2. Use RDBMS Mysql for Database. Initially, Data will be stored in Mysql. We will use Sqoop for moving (import) the data from RDBMS into HADOOP or HIVE. Then we will use Oozie for scheduling jobs.




Topics covered in Case Study:



SDLC (Software Development Life Cycle) / Agile :

SDLC and Agile methodology will be used. We will be able to identify each of the phases in SDLC (Requirements Analysis, Design, Development, Testing, Implementation, and Maintenance). Which will then be use for information gathering and determine functional and non-functional requirements. Requirements will be prioritized to meet customer needs.



Core Java OOP (Object Oriented Programming) concept:

OOPs concepts will be applied here. A business like logic presentaional layer will be developed, with a data access layer for Customers. We will also use JDBC, ODBC drivers for the connection of RDBMS.



Database (Mysql):

We will apply / use RDBMS concept with Structured Query Language (SQL). It will create a Database for the credit card management system and will perform multiple actions in the database such as create, delete, select, insert, update, modify, drop, and commit. We will then connect to the Database with java code/business logic which is defined above using any particular driver.



Data modeling:

We will use Data modeling concepts for Optimization and Normalization in order to eliminate redundant information from the tables to organize data.



Hadoop / HDFS (Hadoop Distributed File System):

Here we will apply / use the Hadoop framework. We will set up Hadoop using Ambari. Then work on the Architecture [Namenode(NN), DataNode(DN)]. Accessing, analyzing, and manipulating data in Hadoop.



Ambari:

We will use Hortonworks sandbox for this case study. Ambari makes Hadoop management simple by providing a consistent, secure platform for operational control.



Hive and Partitioning:

Here we will use / apply Hive and partitioning in hive for better performance. We will also create external and internal tables, that will use hive to design batch queries on Hadoop by providing a declarative abstraction layer. Multiple actions will be performed in the database such as create, delete, select, insert, update, modify, drop.



Sqoop:

Sqoop will be used to import data from the relational source to HDFS and vice versa.



Oozie:

In Oozie we will create a Scheduler system to run and manage Hadoop jobs. Oozie is tightly integrated with the Hadoop stack supporting various Hadoop jobs like Hive, Pig, Sqoop, as well as system specific solution like Java.



Linux / Shell:

Linux and shell commands are used because Hadoop / Ambari is running on Linux (specifically, RHEL/CentOS). Shell will perform any task including the setup of the tools needed for this case study. Bash scripting will automate the extraction, transportation, and loading of data.



Data Visualization:

This Visualization tool is for data exploration and visualization. This helps to complete the entire analysis. This approach presents the opportunity to act immediately, based on what’s happening at a moment in time.
